**3. Helper: `parseZipEntry(directory, path)`** — finds a file in the unzipper directory by `path`, reads its buffer, parses as JSON, returns the parsed array. Throws descriptive error if file missing or not valid JSON array.

```typescript
async function parseZipEntry(
  directory: unzipper.CentralDirectory,
  path: string,
): Promise<unknown[]> {
  const entry = directory.files.find(f => f.path === path)
  if (!entry) throw new Error(`Missing file in archive: ${path}`)
  const buf = await entry.buffer()
  const parsed = JSON.parse(buf.toString('utf-8'))
  if (!Array.isArray(parsed)) throw new Error(`${path} is not a JSON array`)
  return parsed
}
```

**4. POST handler — validation phase (per D-04, IMP-02):**
- Call `requireAdmin(req)` — return 401/403 if auth fails
- Extract file from `req.formData()` — return 422 `{ error: 'Geen bestand geupload' }` if no file or not a File instance
- Convert file to Buffer via `Buffer.from(await file.arrayBuffer())`
- Parse ZIP with `unzipper.Open.buffer(buffer)`
- Find and parse `manifest.json` — return 422 `{ error: 'manifest.json ontbreekt in het archief' }` if missing
- Validate `manifest.schemaVersion === CURRENT_SCHEMA_VERSION` — return 422 `{ error: 'Schema versie komt niet overeen. Verwacht: ${CURRENT_SCHEMA_VERSION}, ontvangen: ${manifest.schemaVersion}' }` if mismatch
- Parse all 18 data files into a `Map<string, unknown[]>` keyed by filename — if any file is missing or not a valid JSON array, return 422 with the error message
- All validation errors returned as `Response.json({ error: string }, { status: 422 })`

**5. Duplicate-detection pre-loading (per D-07, D-08, Research Pattern 6):**
After validation passes but BEFORE opening the transaction:
- Load existing users: `db.user.findMany({ select: { id: true, email: true } })` — build `Map<string, string>` of `email.toLowerCase()` -> `id`
- Load existing places: `db.$queryRaw` `SELECT id, name, address FROM places` — build `Map<string, string>` of `${name}::${address}` -> `id`
- Load existing badges: `db.badge.findMany({ select: { id: true, slug: true } })` — build `Map<string, string>` of `slug` -> `id`
- Load existing photos: `db.photo.findMany({ select: { id: true, storageKey: true } })` — build `Map<string, string>` of `storageKey` -> `id`

Store these four lookup maps as local variables: `existingUsersByEmail`, `existingPlacesByKey`, `existingBadgesBySlug`, `existingPhotosByKey`.

**IMPORTANT conventions:**
- Use `remapRequired(map, oldId, 'contextString')` for non-nullable FKs — throws if mapping is missing
- Use `remap(map, oldId)` for nullable FKs — returns null if oldId is null
- All `createdAt`/`updatedAt`/date fields must be wrapped in `new Date(record.field)` since they come from JSON as strings
- Omit the `id` field from all `create()` calls — let the DB generate new IDs via its default
- For junction tables with compound PKs, use `tx.model.createMany({ data: [...], skipDuplicates: true })` after remapping all records

**2. places.json** (per D-08, D-09 — raw SQL for PostGIS, Research Pattern 4):
```typescript
const places = tableData.get('places.json')!
for (const r of places as any[]) {
  const placeKey = `${r.name}::${r.address}`
  const existingId = existingPlacesByKey.get(placeKey)
  if (existingId) {
    idMaps.places.set(r.id, existingId)
    tableStats['places'].skipped++
    continue
  }
  // CRITICAL: ST_MakePoint takes (longitude, latitude) = (lng, lat)
  const [created] = await tx.$queryRaw<Array<{ id: string }>>`
    INSERT INTO places (name, address, location, created_at, updated_at)
    VALUES (
      ${r.name},
      ${r.address},
      ST_SetSRID(ST_MakePoint(${r.location.lng}, ${r.location.lat}), 4326)::geography,
      ${new Date(r.createdAt)},
      ${new Date(r.updatedAt)}
    )
    RETURNING id
  `
  idMaps.places.set(r.id, created.id)
  tableStats['places'].imported++
}
```

## Interaction Contract

### Export Button — Three States

| State | Visual | Behavior |
|-------|--------|----------|
| Idle | `.btn .btn-primary` — "Download Export" label, fully interactive | `fetch('/api/export')` on click; set loading state |
| Loading | Same button with `disabled` attribute, `opacity-50 cursor-not-allowed` (via `.btn`), spinner replaces label text | Button is unclickable; no duplicate requests |
| Error | Button returns to idle state; error message appears below button in `text-sm text-red-600` | Error message is non-modal — no alert dialogs |

Source: D-02 (spinner + disable during generation), existing `.btn` disabled styles in `globals.css`

### Download Trigger

On successful API response, the browser native download dialog handles the file save. The fetch response carries `Content-Disposition: attachment; filename="snackspot-export-{timestamp}.zip"`. No custom download UI needed.

### Sidebar Navigation

Add one new entry to the `NAV_ITEMS` array in `apps/admin/app/dashboard/layout.tsx`:

```
{ href: '/dashboard/export', label: '📦 Export / Import' }
```

Active state styling is handled automatically by the existing `pathname === item.href` check.

## Page Layout Contract

The export page (`/dashboard/export`) follows the same layout pattern as all dashboard pages:

```
<div>
  <h1 class="text-3xl font-bold mb-8">Export / Import</h1>

  <div class="card">
    <h2 class="text-xl font-semibold mb-4">Gegevens exporteren</h2>
    <p class="text-gray-600 mb-6">{explanatory copy}</p>
    <button class="btn btn-primary">{CTA label}</button>
    {error message — conditional}
  </div>

  <div class="mt-6 bg-orange-50 border border-orange-200 rounded-xl p-4">
    {advisory note — what the export contains}
  </div>
</div>
```

No tabs, no accordion, no modal. Phase 2 will add import UI below the export card on the same page.
